# Design: anagrams

## Overview

`anagrams` is a command-line tool that finds all valid English words whose letters
are a subset of the letters in a given input word. For example, given the word
`"triangle"`, it finds words like `"alert"`, `"grail"`, `"liter"`, `"regal"`, etc.

The problem is drawn from *The Pragmatic Programmer, 20th Anniversary Edition*
(Thomas & Hunt), page 150.

## Core Concepts

### Word Signature

A word's *signature* is the string formed by sorting its letters in uppercase.
Two words are anagrams of each other if and only if they share the same signature.

| Word     | Signature |
|----------|-----------|
| `listen` | `EILNST`  |
| `silent` | `EILNST`  |
| `enlist` | `EILNST`  |
| `decide` | `CDDEEI`  |
| `deiced` | `CDDEEI`  |

Implemented in [signature.go](../signature.go): `SignatureOf(word string) string`.

### Signature Map (`sigmap.json`)

The dictionary is pre-processed into a JSON map keyed by signature:

```json
{
  "EILNST": ["ENLIST", "LISTEN", "SILENT", "TINSEL"],
  "CDDEEI": ["DECIDE", "DEICED"],
  ...
}
```

This map is embedded directly into the compiled binary using Go's `//go:embed`
directive ([search.go](../search.go):13), so no external dictionary file is
required at runtime.

### Subsets

To find all words whose letters are contained in the input word, the algorithm
enumerates all non-empty subsets of the input word's characters. For a word of
length *n*, there are 2Ōü┐ ŌłÆ 1 non-empty subsets.

Each subset is generated by treating integer bit patterns from `1` to `2Ōü┐ ŌłÆ 1`
as bitmasks: bit *i* set means include character *i* from the input word.

Implemented in [subsets.go](../subsets.go): `AllSubsets(s string, length int) []string`.
The `length` parameter filters out subsets shorter than the minimum word length.

### Permutations

[permutations.go](../permutations.go) provides a recursive `Permutations(s string) []string`
function. While not used directly by the search path, it generates all orderings
of a string and is part of the public library API.

## Algorithm: Search

`Search(word string, length int) ([]string, error)` in [search.go](../search.go):

1. **Load dictionary** ŌĆö Deserializes the embedded `sigmap.json` into a
   `map[string][]string` (signature ŌåÆ word list). This is launched as a goroutine
   and the result is communicated back over a channel.
2. **Generate subsets** ŌĆö Calls `AllSubsets(word, length)` to enumerate every
   character subset of the input word that meets the minimum length.
3. **Signature lookup** ŌĆö For each subset, computes its signature and looks it up
   in the map. Any matching dictionary words are collected into an output set
   (a `map[string]int` used to deduplicate).
4. **Sort and return** ŌĆö Converts the output set to a sorted `[]string`.

The use of a goroutine for dictionary loading is a design point: the load happens
concurrently with the subset generation, though the current implementation waits
on the channel before proceeding to lookup.

## Data Preparation: `setup`

The `setup/main.go` tool is a one-time utility that produces `sigmap.json`. It:

1. Reads `words.txt` (a plain-text word list, one word per line).
2. Computes `SignatureOf(word)` for each word.
3. Builds the `map[string][]string` (signature ŌåÆ words).
4. Marshals it to JSON and writes to stdout.

The output is checked into the repository as `sigmap.json` and embedded at
compile time. End users do not need to run `setup` or have `words.txt`.

## CLI: `anagrams`

Entry point: [anagrams/main.go](../anagrams/main.go).

```
usage: anagrams [OPTION]... [WORD]
Finds anagrams of all subsets of the specified word of sufficient length.

options:
  -h, --help               display this help text and exit
  -n, --length=NUM         limits output to words of length >= NUM (default=3)
  -f, --full               include only words the same length as the input
```

- `-n` / `--length` sets the minimum anagram length (default 3).
- `-f` / `--full` restricts output to words exactly as long as the input word
  (equivalent to `-n len(word)`).
- Results are written to stdout, one word per line, in alphabetical order.

Install with:

```
go install github.com/philhanna/anagrams/anagrams@latest
```
